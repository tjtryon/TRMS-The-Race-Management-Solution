What is TRMS?

TRMS brings together four tightly integrated solutions so race-day runs smoothly and your data stays in sync. Use the GUI launcher or console tools to orchestrate every part of your event from a single place.

  • Unified experience: Launch and monitor all components with a GTK4 GUI or console.
  • Cloud or local: Automatic database failover via TRDS keeps you online.
  • Open & extensible: Modular pieces you can adopt individually or as a complete stack.

How it works on race day

  1. Set up your races and entries in TRRS.
  2. Open TRTS to configure chip/manual timing and start the event.
  3. As results are recorded, publish them to TRWS for athletes and spectators.
  4. All data flows through TRDS for reliability and simple backups.

Prefer the terminal? Use the console launchers for automation and scripting.

FAQ

Does TRMS support chip timing?

Yes. TRTS supports RFID chip timing as well as manual workflows, including spacebar capture and later bib assignment.

Can I run the database in the cloud?

Absolutely. TRDS can connect to cloud or local databases and will automatically fail over when configured.

Is there a web results site?

TRWS provides a unified web interface with live results and admin tools to finalize/certify results.

Is it open source?

Yes. TRMS components are open source under the MIT License; see the repositories for details.
